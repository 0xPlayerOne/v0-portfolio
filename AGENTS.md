# AGENTS.md — v0-portfolio

## What this is

Personal portfolio site for Andrew Mahoney-Fernandes. Generated by [v0.dev](https://v0.dev) and deployed on Vercel. Auto-syncs from v0 chats.

- Live: https://vercel.com/nifty-andy/v0-portfolio
- v0 project: https://v0.dev/chat/projects/UNx27p7EMON

## Stack

- **Framework**: Next.js 15.5.18 (App Router) + React 19.2.7
- **Language**: TypeScript 5.9.3, `strict: true`, path alias `@/*` → `./*`
- **Styling**: Tailwind CSS 3.4.19, shadcn/ui (Radix primitives), `lucide-react` icons
- **Package manager**: Bun 1.3.14 (frozen lockfile committed)
- **Toolchain**: `mise.toml` pins node `24.18.0` + bun `1.3.14`; CI uses `jdx/mise-action@v2`
- **Tests**: `bun:test` (built-in) + `@testing-library/react` + `happy-dom` (via `@happy-dom/global-registrator`)
- **Lint/format**: ESLint flat config (`next/core-web-vitals` + `next/typescript`), Prettier 3.9 with `prettier-plugin-tailwindcss`
- **Pre-commit**: husky + lint-staged (prettier + eslint on changed files)

## Scripts (from package.json)

```bash
bun run dev            # next dev
```

CI runs `format:check` → `lint` → `typecheck` → `build` → `test` (job `test` depends on `quality`).

## Code conventions

- **No code comments** unless explicitly requested. Existing comments (e.g. `// Memoize the icon map...`) are leftovers from v0 — leave or remove at author discretion, but do not add new ones.
- **Prettier**: no semis, single quotes, trailing comma `es5`, 100-col, tailwind plugin
- **ESLint**: `_`-prefixed unused vars/args are allowed; `any` and unused vars are relaxed in `*.test.{ts,tsx}` and `tests/**`
- **`next.config.mjs`**: `eslint.ignoreDuringBuilds: true` and `typescript.ignoreBuildErrors: true` — the build does not enforce lint/types. CI enforces them separately, so do not rely on `bun run build` alone to catch issues. Always run `bun run lint` and `bun run typecheck` after changes.
- **shadcn/ui** components live in `components/ui/`. Add new primitives via `bunx --bun shadcn@latest add <name>` rather than hand-writing; do not modify their internals.
- **`cn()` from `@/lib/utils`** (clsx + tailwind-merge) is the standard class-merging helper.
- **Client components**: anything interactive (hooks, state, refs) needs `'use client'` at the top (see `views/about-section.tsx`, all of `components/header/`).
- **Path alias**: import via `@/...`, never relative paths beyond the same directory.
- **Tailwind**: dark mode via `next-themes`; tokens defined in `app/globals.css`. Brand color tokens live in `constants/colors.ts` and `tailwind.config.js`.
- **External data**: `lib/github.ts` fetches with `revalidate: 3600` and falls back to constants from `constants/github.ts` on failure. Never hardcode fallback data inside components — extend the constants.

## Testing

- Test files: `*.test.ts`, `*.test.tsx`, or under `tests/`
- Preload order (see `bunfig.toml`): `tests/dom-globals.ts` (registers `document`/`navigator` globally) → `tests/setup.ts` (RTL `cleanup`, `IS_REACT_ACT_ENVIRONMENT`, `requestAnimationFrame` shim, `mock.restore()`)
- DOM env: `bun test --dom` with `happy-dom`; `@testing-library/react` v16
- New tests are picked up automatically; no config changes needed

## Environment

- Local env in `.env.local` (gitignored). Pull from Vercel: `vercel link && vercel env pull .env.local`. Push back: `vercel env push .env.local`.
- Never commit `.env.local` or any tokens. CI does not need them.

## When you make changes

1. Run `bun run format:check && bun run lint && bun run typecheck` locally before considering work done. (CI blocks on these.)
2. If you changed behavior covered by tests, run `bun run test`.
3. If you added deps, keep `bun.lock` committed and use `bun install --frozen-lockfile` in CI-shaped commands.
4. Do not commit unless explicitly asked. Do not amend or force-push.
5. Match the surrounding file's style (e.g. `'use client'` placement, named exports, import order).
